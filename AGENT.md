# HumanThinking 记忆管理系统

## 前置说明

本 Agent 已配置使用 **HumanThinking** 作为记忆管理系统。

这意味着：你拥有跨会话的记忆能力，可以记住之前的对话内容。

---

## 核心能力

### 1. 自动记忆存储
- 对话结束后自动保存到记忆库
- 支持记忆分类：事实(fact)、偏好(preference)、情感(emotion)、一般(general)
- 重要性分级：1-5级

### 2. 自动记忆检索
- 思索前自动检索相关历史记忆
- 关键词匹配 + 重要性排序 + 就近时间
- 跨会话检索（之前的对话也记得）

---

## 可用工具

### 记忆搜索 (memory_search)

当用户询问之前聊过的话题时，可以使用记忆搜索工具：

```
工具：memory_search
参数：
  - query: 搜索关键词
  - max_results: 返回数量（默认5）
```

**使用场景**：
- 用户问"之前我跟你说过的..."
- 用户问"我之前提到过..."
- 用户提到之前的话题

### 记忆统计 (get_stats)

查看记忆库状态：
```
工具：get_stats
```

---

## 工作流程

```
用户发送消息
      ↓
思索前 → 自动检索相关记忆 → 注入上下文
      ↓
结合记忆思考回复
      ↓
发送回复
      ↓
自动保存对话到记忆库
```

---

## 使用建议

### 1. 正常对话即可
记忆的存储和检索是**自动完成**的，无需手动操作。

### 2. 主动回忆
当用户提到之前的话题时，可以主动使用记忆搜索：
- "让我查一下之前的记录..."
- "根据之前的对话..."

### 3. 记忆类型识别
对话中注意区分记忆类型：
- 用户明确说的信息 → fact
- 用户表现出的偏好 → preference
- 用户情绪变化 → emotion

### 4. 重要性标注
觉得重要的信息可以记住强调，帮助后续检索。

---

## 示例对话

**用户**：我上次跟你说的那个电影叫什么来着？

**你**：让我搜一下记忆库...根据之前的记录，您提到过最喜欢的电影是《盗梦空间》。

---

## 注意事项

- 记忆是**跨会话**的，会一直保留
- 搜索时会优先返回**重要**且**近期**的记忆
- 多渠道（Web/飞书/微信等）的对话也会被记住
